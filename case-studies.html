We don't talk about what we can do — we show what we have done. These are selected examples from 15+ years of enterprise delivery across Norway and Europe.

01 — Public Sector / Cloud Migration

Oracle ExaDB 19c → 26ai Migration

Norwegian Tax Authority (Skatteetaten) · Azure / Oracle Cloud

Oracle ExaDB Azure Zero downtime
Challenge

Legacy Oracle 19c database running critical national tax processing — required migration to ExaDB 26ai on Azure without any downtime or data loss.

Approach

Designed a Data Pump export pipeline with parallel streams, OCI Object Storage staging, and automated cutover orchestration using Azure VM run-commands.

Outcome

Zero-downtime migration completed. Full schema and data integrity verified. Processing performance improved 40% on the new ExaDB 26ai cluster.

0Minutes downtime
40%Performance uplift
100%Data integrity verified

02 — Healthcare / Data Platform

National Health Data Platform

Norwegian Health Authority · Azure · DIPS Arena

Azure Data Engineering Healthcare
Challenge

Clinical data siloed across disparate hospital systems — no unified view for analytics, reporting, or AI-assisted clinical decision support.

Approach

Built an Azure-native data lakehouse with medallion architecture. FHIR-compliant ingestion pipelines, dbt transformations, and a semantic layer for BI and ML consumption.

Outcome

Unified clinical data platform serving multiple hospital trusts. Enabled first national-scale analytics on patient pathway data, cutting reporting cycle from weeks to hours.

03 — ERP / Manufacturing

D365 F&O Enterprise Programme — Alfa Laval

Alfa Laval · Microsoft Dynamics 365 · Europe-wide

D365 F&O Azure Integration Multi-country
Challenge

One of the largest D365 implementations in Europe — upgrading global ERP across multiple legal entities, countries, and thousands of users while maintaining business continuity.

Approach

Phased country rollout with parallel-run periods. X++ customisations for local compliance. Azure Data Factory integration layer connecting D365 to supply chain and logistics systems.

04 — AI / Developer Tooling

Nexus — Multi-Model AI Coding Agent

Adaptiv AS internal · Claude · GPT-4 · Gemini

AI / LLM Python · Streamlit Open Source
Challenge

AI coding assistants are expensive and single-model — no intelligent routing across Claude, GPT-4, and Gemini based on task type and cost optimisation.

Approach

Built a keyword-based routing layer that dispatches tasks to the optimal model — free models preferred on ties, paid models for quality-critical operations. Streaming Streamlit UI.

Outcome

70%+ of tasks handled by free models without quality loss. Full tool suite: browser automation, GitHub, database migration, code execution, and memory.
